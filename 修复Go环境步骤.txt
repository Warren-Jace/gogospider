【紧急】Go环境修复步骤
=============================

问题：Go标准库文件缺失，无法编译
原因：Go安装不完整（可能是下载中断或解压错误）

✅ 推荐方案：重新安装Go
-------------------------

步骤1：下载完整的Go安装包
  访问：https://go.dev/dl/
  或直接下载：https://go.dev/dl/go1.23.6.windows-amd64.msi
  文件大小：约67MB（确保下载完整）

步骤2：卸载旧版本
  1. 打开"控制面板" -> "程序和功能"
  2. 找到 "Go Programming Language"
  3. 右键卸载
  
  或者直接删除目录：
    - 删除：D:\Env\go1.23.6.windows-amd64\go
    - 删除：C:\Users\hacker\go (如果存在)

步骤3：安装新版本
  1. 双击下载的 go1.23.6.windows-amd64.msi
  2. 选择安装路径（建议默认 C:\Go）
  3. 完成安装
  4. 重启命令行窗口

步骤4：验证安装
  打开新的PowerShell窗口，运行：
    go version
    go env GOROOT
  
  应该显示：
    go version go1.23.6 windows/amd64
    C:\Go (或您选择的路径)

步骤5：编译程序
  cd C:\Users\hacker\Desktop\test\gogospider
  go build -o spider_fixed.exe cmd\spider\main.go

⚠️ 备用方案：手动修复当前安装
-----------------------------

如果无法重新安装，尝试：

1. 重新下载zip版本：
   https://go.dev/dl/go1.23.6.windows-amd64.zip

2. 完全删除旧目录：
   Remove-Item -Recurse -Force "D:\Env\go1.23.6.windows-amd64\go"

3. 解压到相同位置
   - 使用7-Zip或WinRAR
   - 确保完整解压（不要中断）

4. 验证文件：
   Test-Path "D:\Env\go1.23.6.windows-amd64\go\src\runtime\runtime.go"
   应该返回：True

5. 设置环境变量（如果需要）：
   $env:GOROOT = "D:\Env\go1.23.6.windows-amd64\go"
   $env:PATH = "$env:GOROOT\bin;$env:PATH"

🚀 快速测试
-----------

安装完成后，测试一个简单程序：

1. 创建test.go：
   echo 'package main
   import "fmt"
   func main() { fmt.Println("Hello") }' > test.go

2. 编译测试：
   go build test.go

3. 如果成功，继续编译spider：
   go build -o spider_fixed.exe cmd\spider\main.go

3. dir "GOROOT路径\src\runtime" 的输出
